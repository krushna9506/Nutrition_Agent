# 🥗 NutriAI Agent
### AI-Powered Nutrition Assistant — IBM Watsonx.ai × Granite

> A full-stack web application that delivers personalized nutrition plans, calorie analysis,
> BMI tracking, Indian meal recommendations, and family diet management — powered by **IBM Watsonx.ai Granite** models.

## ✨ Features

| Feature | Description |
|---|---|
| 💬 **AI Chat** | Conversational nutrition agent with persona, Indian food knowledge & safety rules |
| 📊 **Nutrition Dashboard** | BMR/TDEE calculator with macro breakdown (protein, carbs, fats) |
| 🗓️ **Meal Planner** | 1–7 day personalized Indian meal plans generated by Granite AI |
| ⚖️ **BMI Calculator** | BMI scale, category, ideal weight range + AI personalized advice |
| 👨‍👩‍👧‍👦 **Family Profiles** | Add up to 8 family members, get a unified family nutrition plan |
| 🔍 **Food Analyzer** | Instant calorie + macro analysis for any Indian or international dish |
| 🌙 **Dark Mode** | Persisted theme toggle with smooth transitions |
| 📱 **Mobile Responsive** | Full Bootstrap 5 responsive layout |

## 🚀 Quick Start

### 1. Prerequisites

- Python 3.9 or higher
- An IBM Cloud account ([Sign up free](https://cloud.ibm.com/registration))
- An IBM Watsonx.ai project ([Get started](https://dataplatform.cloud.ibm.com/wx/home))

### 2. Get IBM Cloud Credentials

**Step A — Create an API Key:**
1. Go to [IBM Cloud IAM → API Keys](https://cloud.ibm.com/iam/apikeys)
2. Click **Create an IBM Cloud API key**
3. Name it (e.g., `nutriai-key`) and copy the key immediately

**Step B — Get your Watsonx Project ID:**
1. Open [IBM Watsonx.ai](https://dataplatform.cloud.ibm.com/wx/home)
2. Open your project → **Manage** tab → **General**
3. Copy the **Project ID** (UUID format)

### 3. Configure Environment

```bash
# Copy the example env file
cp .env.example .env

# Edit .env and add your credentials
notepad .env          # Windows
nano .env             # Linux/macOS
```

Set these values in `.env`:
```env
IBM_CLOUD_API_KEY=your_actual_api_key
WATSONX_PROJECT_ID=your_actual_project_id
WATSONX_URL=https://us-south.ml.cloud.ibm.com
```

### 4. Install Dependencies

```bash
# Create a virtual environment (recommended)
python -m venv venv

# Activate it
venv\Scripts\activate        # Windows
source venv/bin/activate     # Linux/macOS

# Install packages
pip install -r requirements.txt
```

### 5. Run the Application

```bash
python app.py
```

Open your browser at: **http://localhost:5000**

## 🔧 Customizing the Agent (IMPORTANT)

All agent behavior is controlled in **`agent_instructions.py`** — no other files need editing.

### Sections you can customize:

| Section | What to change |
|---|---|
| `PERSONA` | Name, tone, communication style |
| `EXPERTISE` | Add/remove health domains |
| `INDIAN_FOOD_PREFERENCES` | Regional cuisines, festivals, measurements |
| `DIET_SPECIALIZATIONS` | Add new diet types (e.g., Mediterranean, GAPS) |
| `RESPONSE_FORMAT` | How AI structures its responses |
| `SAFETY_RULES` | Disclaimer rules, banned advice |
| `FAMILY_INSTRUCTIONS` | Family plan behavior |
| `CALORIE_FORMULAS` | Override formulas if needed |

**Example — Change the agent name and make it more casual:**
```python
# In agent_instructions.py
PERSONA = """
You are Mitali, a fun and friendly nutrition coach from Mumbai.
You use simple language, mix a little Hindi (like 'bilkul!' and 'arre!'),
and make healthy eating feel exciting and achievable...
"""
```

## 🌐 API Reference

All endpoints return JSON. Frontend JavaScript calls these automatically.

### `POST /api/chat`
Conversational AI chat.
```json
Request:  { "message": "...", "history": [...], "profile": {...} }
Response: { "response": "...", "timestamp": "..." }
```

### `POST /api/bmi`
Calculate BMI.
```json
Request:  { "weight_kg": 70, "height_cm": 170 }
Response: { "bmi": 24.2, "category": "Normal Weight", "advice": "..." }
```

### `POST /api/calories`
Calculate daily calorie needs.
```json
Request:  { "weight_kg": 70, "height_cm": 170, "age": 30,
            "gender": "male", "activity": "moderate", "goal": "lose" }
Response: { "bmr": 1750, "tdee": 2712, "target_calories": 2312,
            "protein_g": 156, "carbs_g": 260, "fats_g": 72 }
```

### `POST /api/meal-plan`
Generate AI meal plan.
```json
Request:  { "profile": {...}, "days": 7, "preferences": "..." }
Response: { "meal_plan": "...", "days": 7 }
```

### `POST /api/family-plan`
Generate family nutrition plan.
```json
Request:  { "members": [{ "name": "Rahul", "age": 35, ... }] }
Response: { "family_plan": "...", "member_count": 4 }
```

### `POST /api/analyze-food`
Analyze food nutritional content.
```json
Request:  { "food_description": "2 idli with sambhar" }
Response: { "analysis": "...", "food": "..." }
```

### `GET /health`
Health check / status.
```json
Response: { "status": "ok", "watsonx_model": "configured" }
```

## 🐳 Docker Deployment

```bash
# Create a Dockerfile
cat > Dockerfile << 'EOF'
FROM python:3.11-slim
WORKDIR /app
COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt
COPY . .
EXPOSE 5000
CMD ["gunicorn", "--bind", "0.0.0.0:5000", "--workers", "2", "app:app"]
EOF

# Build and run
docker build -t nutriai-agent .
docker run -p 5000:5000 --env-file .env nutriai-agent
```

## ☁️ IBM Cloud Code Engine Deployment

```bash
# Install IBM Cloud CLI + Code Engine plugin
ibmcloud login --apikey $IBM_CLOUD_API_KEY
ibmcloud ce project select --name nutriai-project

# Create secrets
ibmcloud ce secret create --name nutriai-secrets \
  --from-literal IBM_CLOUD_API_KEY=$IBM_CLOUD_API_KEY \
  --from-literal WATSONX_PROJECT_ID=$WATSONX_PROJECT_ID

# Deploy

  --name nutriai-agent \
  --image icr.io/your-namespace/nutriai-agent:latest \
  --env-from-secret nutriai-secrets \
  --port 5000
```

## 🔒 Security Notes

- `.env` is listed in `.gitignore` — never commit it
- In production, set `FLASK_DEBUG=False`
- Change `FLASK_SECRET_KEY` to a cryptographically random value:
  ```python
  python -c "import secrets; print(secrets.token_hex(32))"
  ```
- Use environment variables or IBM Secrets Manager in production — not `.env` files

## 🐛 Troubleshooting

| Problem | Solution |
|---|---|
| `IBM_CLOUD_API_KEY not configured` | Edit `.env` and add your real key |
| `Failed to initialize Watsonx.ai` | Check your Project ID and API key; verify Watsonx is enabled in your region |
| `ModuleNotFoundError: ibm_watsonx_ai` | Run `pip install -r requirements.txt` in your virtual env |
| App shows fallback responses | AI works in demo mode without a key — configure `.env` for full AI |
| Port already in use | Change `FLASK_PORT=5001` in `.env` |
